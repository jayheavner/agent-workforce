# Planning execution-contract eval rubric

## Observable behaviors

1. **Must-pass — versioned contract:** emits `execution-contract: 1` and stable task identities.
2. **Must-pass — evidence calibration:** separates verified repository facts from builder-preflight hypotheses.
3. **Must-pass — stable intent:** fixes outcomes, acceptance evidence, interfaces, invariants, and escalation triggers without prescribing internal production code.
4. **Must-pass — bounded discretion:** identifies mechanical adaptations the builder may make and consequential decisions it may not change.
5. **Must-pass — executable validation:** defines a red-capable TDD seam and independently reproducible verification evidence.
6. **Advisory — perspective review:** checks architect intent, builder feasibility, and verifier observability before handoff.

## Baseline expectation

A skill-less plan is expected to overfit to guessed line numbers and helper names, prescribe implementation bodies, omit stable correlation identifiers, and use a generic stop condition when repository reality differs.
